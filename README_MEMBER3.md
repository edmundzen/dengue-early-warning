# Member 3 – Decision Intelligence Dashboard & UI/UX Presentation

## ## Overview
This component represents the Frontend Application tier of the Dengue Cluster Early-Warning System. It translates backend data engineering (Member 1) and predictive risk modeling (Member 2) into an enterprise-grade, user-facing **Decision Intelligence Platform** via Google Looker Studio. This interface empowers government health departments, epidemiologists, and field workers to detect potential dengue clusters early, understand risks at a glance, and enable timely decision-making for prevention.

## Input

BigQuery Dataset:
- dengue_ew

Input Tables:
- features
- inspection_priority (Generated by Member 2)

Input Features:
- cell_id
- lat
- lon
- risk_score / case_density_14d

---

## Processing Steps

1. Connect Google Looker Studio to the Google BigQuery data warehouse.
2. Ingest the prioritized risk tables and telemetry.
3. Configure a Google Maps interactive heatmap utilizing lat/lon and risk scores.
4. Generate a descending "Top-10 High-Risk" triage table for field worker action.
5. Apply conditional formatting rules (Red >= 25, Amber 11-24, Green <= 10).
6. Implement executive KPI scorecards tracking maximum risk and total monitored cells.
7. Apply strict UI/UX design branding (Poppins font, #114232 header).
