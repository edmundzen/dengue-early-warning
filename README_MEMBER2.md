# Member 2 – GPU Risk Model

## Overview
This notebook generates dengue inspection priorities by combining engineered features from Member 1 with the latest 14-day forecast generated by the forecasting pipeline. It computes a GPU-accelerated risk score for every geographic cell, ranks locations based on outbreak risk, assigns risk levels, and stores the final inspection priority table in Google BigQuery.

## Inputs

### BigQuery Dataset
`dengue_ew`

### Input Tables
- `features`
- `forecast_14d`

### Input Features
- case_density_14d
- rain_lag_7to14d
- recurrence
- forecast_value

## Processing Steps
1. Connect to Google BigQuery.
2. Load the engineered feature table.
3. Load the latest forecast for each geographic cell.
4. Merge features with forecast values.
5. Compute a GPU-accelerated dengue risk score.
6. Rank all locations by descending risk score.
7. Assign High, Medium, and Low risk levels.
8. Create the inspection priority table.
9. Upload the final results back to BigQuery.

## Output

### BigQuery Table
`inspection_priority_v2`

### Columns
- rank
- cell_id
- lat
- lon
- date
- risk_score
- risk_level

## Technologies Used
- Python
- RAPIDS cuDF
- Pandas
- Google BigQuery
- Google Colab
- NVIDIA T4 GPU

## Result
The notebook successfully generates inspection priorities using engineered features and forecast outputs, then uploads the final ranked table to:

`dengue_ew.inspection_priority_v2`

The output is ready for dashboard visualization, inspection planning, and operational decision-making.
