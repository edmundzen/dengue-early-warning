# Member 2 – GPU Risk Model

## Overview
This notebook generates dengue inspection priorities by combining engineered features from Member 1 with the latest 14-day forecast generated by the forecasting pipeline. It computes a GPU-accelerated risk score for every geographic cell, ranks locations based on outbreak risk, assigns risk levels, creates the alert queue for high-priority locations, and stores the final outputs in Google BigQuery.

---

## Inputs

### BigQuery Dataset
`dengue_ew`

### Input Tables
- `features`
- `forecast_14d`

### Input Features
- `case_density_14d`
- `rain_lag_7to14d`
- `recurrence`
- `forecast_value`

---

## Processing Steps

1. Connect to Google BigQuery.
2. Load the engineered feature table.
3. Load the latest forecast for each geographic cell.
4. Merge engineered features with forecast values.
5. Compute a GPU-accelerated dengue risk score.
6. Rank all locations by descending risk score.
7. Assign risk levels (Low, Medium, High, Critical).
8. Create the `inspection_priority_v2` table.
9. Upload the inspection priority table to BigQuery.
10. Create the `v_daily_trend` view for dashboard trend visualization.
11. Generate the `alert_queue` table containing High and Critical risk locations for downstream alerting.
